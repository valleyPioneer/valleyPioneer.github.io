C++和C中const的区别

发表于 2018-04-26 | 分类于 C/C++

C++

  • C++中可以使用const完全取代#define
  • const属于常量

C

  • C中const作用不大，多用#define
  • const属于不能改变的普通变量

主要区别

  1. c中是变量，总是占用内存，而且属于全局变量，而c++中是常量

  2. c中默认const是外部连接，c++默认是内部连接

内部连接和外部连接

  • 内部连接

    假如一个名称对于他的编译单元来说是局部的，并且在连接时不会和其他编译单元中的同样的名称相冲突，那么这个名称有内部连接。

  • 外部连接

    在一个多文件程序中,如果一个名称在连接时可以和其他编译单元交互,那么这个名称就具有外部连接。具有外部连接对应的名称在程序中必须是唯一的。

Android优化启动速度避免出现黑屏或者白屏

发表于 2018-04-24 | 分类于 Android

黑白屏其实是预览窗口，Preview Window。Android系统会根据你的manifest文件设置的主题颜色的不同来展示一个白屏或者黑屏。本身是系统为了提高用户体验而有意设定的。

如何优化app启动速度

  • 在Application的构造器方法、attachBaseContext()、onCreate()方法中不要进行耗时操作的初始化，一些数据预取放在异步线程中，可以采取Callable实现。
  • 对于sp的初始化，因为sp的特性在初始化时候会对数据全部读出来存在内存中，所以这个初始化放在主线程中不合适，反而会延迟应用的启动速度，对于这个还是需要放在异步线程中处理。
  • 对于MainActivity，由于在获取到第一帧前，需要对contentView进行测量布局绘制操作，尽量减少布局的层次，考虑ViewStub的延迟加载策略，当然在onCreate、onStart、onResume方法中避免做耗时操作。

以上三种策略可以真实地提高app的启动速度。

从另一方面考虑，可以修改preview window的显示效果来实现等待第一帧的用户体验。

onSaveInstanceState调用时机

发表于 2018-04-16 | 分类于 Android

当某个activity容易被系统销毁时（具有可能性，但是还没有被销毁）。

  1. 按下home键
  2. 长按home键，选择运行其他程序
  3. 按下电源键（关闭屏幕显示）
  4. 从当前活动启动一个新的活动
  5. 屏幕方向切换时，从竖屏切换至横屏
  6. 电话打入等情况

onRestoreInstanceState调用时机

onSaveInstanceState方法和onRestoreInstanceState方法“不一定”是成对的被调用的，onRestoreInstanceState被调用的前提是，activity A“确实”被系统销毁了，而如果仅仅是停留在有这种可能性的情况下，则该方法不会被调用，例如，当正在显示activity A的时候，用户按下HOME键回到主界面，然后用户紧接着又返回到activity A，这种情况下activity A一般不会因为内存的原因被系统销毁，故activity A的onRestoreInstanceState方法不会被执行。

onRestoreInstanceState方法中bundle参数和onCreate方法中bundle参数相同，都是onSaveInstanceState方法中返回的，因此也可以选择在onCreate中做数据还原。

Android finish方法与异常生命周期

发表于 2018-04-08 | 分类于 Android
  1. 调用finish方法后，activity会从栈中移除，此时实例和资源并没有被销毁。如果按back键，由于已经不再活动栈中，因此无法返回到该活动中。

  2. 异常的生命周期是指Activity被系统回收或者当前设备的Configuration发生变化（一般指横竖屏切换）从而导致Activity被销毁重建。异常的生命周期主要分以下两种情况：

    1. 相关的系统配置发生改变导致Activity被杀死并重新创建（一般指横竖屏切换）
    2. 内存不足导致低优先级的Activity被杀死
    3. 两者对应的生命周期均为当我们正常启动Activity时，onCreate，onStart，onResume方法都会依次被回调，而如果我们此时把竖屏的Activity人为的调整为横屏，我们可以发现onPause，onSaveInstanceState，onStop，onDestroy，onCreate，onStart，onRestoreInstanceState，onResume依次被调用，单从调用的方法我们就可以知道，Activity先被销毁后再重新创建

  3. 在一些突发情况下，系统回收内存会导致onStop、onDestroy方法不被调用，因此onPause方法是保证可以被调用的最后一个生命周期中的方法。

Android apk 反编译总结

发表于 2018-03-19 | 分类于 Android

工具介绍

  • apktool
    作用：主要查看res文件下的xml文件、AndroidManifest文件和图片。（直接解压apk得到的xml文件大多是乱码）

  • dex2jar
    作用：将classes.dex文件转换成jar文件

  • jd-gui
    作用：查看jar文件，将jar文件转换成java文件

  • smali2JavaUI
    作用：直接打开apk，结合以上两步直接得到java源文件或者将smali文件转换成java文件。缺点在于，无法解析出资源文件。

反编译以及重新打包过程中使用到的命令

  1. 反编译apk
    apktool d app.apk

  2. 修改资源文件或者修改smali源代码

  3. 打包
    apktool b app -o app_new.apk

  4. 签名
    mykeystore.jks^1:使用Android Studio生成

    签名命令：jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 -keystore mykeystore.jks -storepass android -signedjar app_new_signed.apk app_new.apk mykeystore.jks

  5. 对齐
    zipalign 4 app_new_signed.apk app_new_signed_aligned.apk
